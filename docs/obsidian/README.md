# HIIT Master Vault

This folder is a reverse-engineered, Obsidian-style product/design vault for the current HIIT Master app.

The goal is not to document every implementation detail. The goal is to capture enough product intent, screen behavior, and domain rules that a fresh implementation could be produced faster and with less UI/code thrash.

Conventions:

- `Current behavior` means the app behaves this way now.
- `Design intent` means this is the product rule the implementation should preserve.
- `Open issue` means the current design is incomplete or inconsistent.
